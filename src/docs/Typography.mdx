# Typography

Inter font family with a carefully designed type scale for clear hierarchy. JetBrains Mono for code and technical content.

---

## Display Scale

Heading hierarchy for page titles and section headers.

| Name | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Display XL | 64px | 700 | 1.1 | `text-6xl font-bold` |
| Display Large | 48px | 700 | 1.2 | `text-5xl font-bold` |
| Display Medium | 36px | 700 | 1.3 | `text-4xl font-bold` |
| Display Small | 24px | 600 | 1.4 | `text-2xl font-semibold` |

## Body Scale

Text styles for paragraphs and supporting content.

| Name | Size | Weight | Line Height | Use Case |
|------|------|--------|-------------|----------|
| Body Large | 18px | 400 | 1.75 | Lead paragraphs and important text |
| Body Medium | 16px | 400 | 1.75 | Standard body text for content |
| Body Small | 14px | 400 | 1.5 | Captions and metadata |

## Text Colors

- **Primary text** (`text-foreground`) — headings and emphasis
- **Muted text** (`text-muted-foreground`) — body copy and descriptions
- **Subtle text** (`text-subtle`) — captions and metadata

## Monospace

JetBrains Mono for code and technical content.

```tsx
<p className="font-mono text-sm">const theme = "atelier-dark";</p>
<code className="rounded-md bg-muted px-3 py-1.5 font-mono text-sm text-primary">
  npm install @atelier/design-tokens
</code>
```

## Responsive Typography

| Breakpoint | Hero Size | Section Size |
|-----------|-----------|-------------|
| Mobile (< 768px) | `text-4xl` | `text-3xl` |
| Tablet (768px–1024px) | `text-5xl` | `text-4xl` |
| Desktop (> 1024px) | `text-6xl` | `text-4xl` |

---

## Best Practices

**Do:**
- Use Display XL sparingly — only for hero sections or landing pages
- Apply `tracking-tight` to display headings for a polished look
- Use `text-muted-foreground` for body text and `text-foreground` for headings
- Use the monospace font for code, technical content, and data labels

**Don't:**
- Don't skip heading levels (e.g., h1 → h3 without h2)
- Avoid using more than 2 font weights on a single page
- Don't set line-height below 1.4 for body text — it hurts readability
- Never use custom font sizes outside the defined scale

## Installation

```sh
# Self-host fonts from /public (served at /fonts/...)
# Expected structure in this repo:
# - public/fonts/Inter/web/*.woff2
# - public/fonts/JetBrainsMono/fonts/webfonts/*.woff2
```

```css
@font-face {
  font-family: 'Inter';
  font-style: normal;
  font-weight: 400;
  font-display: swap;
  src: url('/fonts/Inter/web/Inter-Regular.woff2') format('woff2');
}

@font-face {
  font-family: 'JetBrains Mono';
  font-style: normal;
  font-weight: 400;
  font-display: swap;
  src: url('/fonts/JetBrainsMono/fonts/webfonts/JetBrainsMono-Regular.woff2') format('woff2');
}
```
